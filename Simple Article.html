Способы хранения данных ﻿в
ОС Android
Обзор возможностей для хранения данных в ОС Android. Использование файловой системы. Создание и управление локальными базами данных. Инструменты SQLite. Типы данных SQLite. Использование адаптеров для отображения данных
   Приложение, написанное по требованиям технического задания, реализует различные способы хранения данных. К ним относятся: хранение данных в файле (во внутреннем хранилище и на SD-карте), в базе данных и с использованием SharedPreferences.
   Приложение создано на основе шаблона EmptyActivity. Автоматически создаётся Java-класс приложения (MainActivity.java), в котором описывается поведение элементов графического интерфейса, и непосредственно форма главного окна (activity_main.xml).
   Для начала необходимо добавить на форму компоненты, которые будут использоваться в приложении. Это можно сделать либо с помощью дизайнера, где необходимо перетягивать какой-либо элемент на форму, при этом xml-код будет автоматически сгенерирован в файле activity_main.xml. Также возможен вариант написания xml-кода вручную.

     Далее будет рассмотрена реализация различных способов хранения данных:

1. Хранение во внутреннем хранилище.
     Используя данный способ данные из компонента EditText будут записаны в файл, который будет создан в папке приложения по пути /data/data/namefile.txt. Запись в файл будет осуществляться по нажатию кнопки. Данная функция реализована следующим программным кодом:
public void SaveFile(View view) {
   String FILE_NAME = "file.txt";
   FileOutputStream fos = null;
   try {
       fos = openFileOutput(FILE_NAME, MODE_APPEND);
       fos.write(editTextEnter.getText().toString().getBytes());
       Toast.makeText(this, "Файл сохранён", Toast.LENGTH_SHORT).show();
   }
   catch(IOException ex) {
       Toast.makeText(this, ex.getMessage(), Toast.LENGTH_SHORT).show();
   }
   finally{
       try{
           if(fos!=null)
               fos.close();
       }
       catch(IOException ex){

           Toast.makeText(this, ex.getMessage(), Toast.LENGTH_SHORT).show();
       }
   }
   editTextEnter.setText("");
}
     При записи в файл могут возникнуть ошибки, которые будут обрабатываться в блоке try-catch и при успешном сохранение текста в файл на экране появится всплывающее сообщение о том, что файл сохранён. Если всё-таки возникла какая-то ошибка, то на экране также появится сообщение об ошибке.
     При нажатии на кнопку «Открыть файл» будет осуществляться переход на другую Activity(SecondActivity), в которую с помощью использования намерения (Intent) передаётся id нажатой кнопки. В SecondActivity находится TextView для вывода текста из файла. В SecondActivity происходит получение id кнопки из главной Activity и соответствующие действия.
Функция чтения информации из файла представлена ниже:

2. Хранение в базе данных
    Для использования базы данных необходимо реализовать класс DatabaseHelper, который является наследником класса SQLiteOpenHelper. В данном классе необходимо будет переопределить два основных метода onCreate() и onUpgrade(). В методе onCreate() будет выполняться SQL-запрос на создание таблицы, в которой будет храниться введённый текст. Таблица состоит и двух столбцов: id и cur_text. Разработанный класс представлен ниже:
package com.example.datastorage;
import android.content.Context;
import android.database.sqlite.SQLiteDatabase;
import android.database.sqlite.SQLiteOpenHelper;

public class DatabaseHelper extends SQLiteOpenHelper {
   private static final String DATABASE_NAME = "MyDB.db"; // название бд
   private static final int SCHEMA = 2; // версия базы данных
   static final String TABLE = "myText"; // название таблицы в бд
   public static final String COLUMN_ID = "_id"; // названия столбцов
   public static final String COLUMN_CUR_TEXT = "cur_text";
   public DatabaseHelper(Context context) {
       super(context, DATABASE_NAME, null, SCHEMA);
   }
   @Override
   public void onCreate(SQLiteDatabase db) {
       db.execSQL("CREATE TABLE IF NOT EXISTS " + TABLE + "(" + COLUMN_ID + " INTEGER PRIMARY KEY AUTOINCREMENT, "+
               COLUMN_CUR_TEXT + " TEXT);");
   }
   @Override
   public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
       db.execSQL("DROP TABLE IF EXISTS " + TABLE);
       onCreate(db);
   }
}
     Реализация функции записи в БД:
public void SaveInBD(View view) {
   db = databaseHelper.getWritableDatabase(); //Открываем БД для записи
   ContentValues newValues = new ContentValues();//для добавления строки в таблицу
   newValues.put(COLUMN_CUR_TEXT, editTextEnter.getText().toString());//добавляем строку
   db.insert(DatabaseHelper.TABLE, null, newValues); //запрос на добавление в таблицу текста
   Toast.makeText(this, "Текст сохранён в БД", Toast.LENGTH_SHORT).show();
   editTextEnter.setText("");
}
     При нажатии на кнопку «Открыть БД» будет осуществляться переход на другую Activity(SecondActivity), в которую с помощью использования намерения (Intent) передаётся id нажатой кнопки. Реализация функции выборки данных из БД:
SQLiteDatabase db = databaseHelper.getWritableDatabase();
Cursor userCursor = db.rawQuery("SELECT COUNT(*) FROM " + TABLE , null);
userCursor.moveToFirst();
int count = userCursor.getInt(0);
if(count != 0) {
   userCursor = db.rawQuery("SELECT " + COLUMN_CUR_TEXT + " FROM " + TABLE + " WHERE " + COLUMN_ID + " = ? ", new String[]{String.valueOf(count)});
   userCursor.moveToFirst();
   String str = userCursor.getString(0);
   textView2.setText(str);
}
     При открытии БД извлекается последняя запись и помещается в TextView.

3. Использование Preferences, когда значения сохраняются в виде пары: имя, значение.
     Для сохранения данных, сначала с помощью метода getPreferences получаем объект sPref класса SharedPreferences, который позволяет работать с данными (читать и писать). Константа MODE_PRIVATE используется для настройки доступа и означает, что после сохранения, данные будут видны только этому приложению.

     Функция сохранения данных:
SharedPreferences sPref;
sPref = getPreferences(MODE_PRIVATE);
SharedPreferences.Editor ed = sPref.edit();
ed.putString(SAVED_TEXT, editTextEnter.getText().toString());
ed.commit();
Toast.makeText(this, "Текст сохранён", Toast.LENGTH_SHORT).show();
editTextEnter.setText("");
     При загрузке данных с помощью метода getPreferences получаем объект sPref класса SharedPreferences. MODE_PRIVATE снова указывается, хотя и используется только при записи данных. Здесь Editor не используется, т.к. необходимо выполнить только чтение данных. Читаем с помощью метода getString – в параметрах указываем константу – это имя, и значение по умолчанию (пустая строка). Далее считанные данные передаются через Intent в другую Activity.
sPref = getPreferences(MODE_PRIVATE);
String savedText = sPref.getString(SAVED_TEXT, "");
Intent intent = new Intent(MainActivity.this, SecondActivity.class);
intent.putExtra("idButton", view.getId());
intent.putExtra("spref", savedText);
startActivity(intent);

4. Использование SD-карты.
     При использовании данного способа, необходимо прописать в файле AndroidManifest.xml разрешение на использование SD-карты.
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"

     Замечание! При использовании версии Android выше 6 необходимо реализовывать запрос у пользователя на использование SD-карты.


     Реализация функции записи на SD:
public void writeFileSD(View view) {
   if (!Environment.getExternalStorageState().equals( // проверяем доступность SD
           Environment.MEDIA_MOUNTED)) {
       Toast.makeText(this, "SD-карта не доступна: " + Environment.getExternalStorageState(), Toast.LENGTH_SHORT).show();
       return;
   }
  File sdPath = Environment.getExternalStorageDirectory(); // получаем путь к SD
   sdPath = new File(sdPath.getAbsolutePath() + "/" + DIR_SD); // добавляем свой каталог к пути
   sdPath.mkdirs(); // создаем каталог
   File sdFile = new File(sdPath, FILENAME_SD); // формируем объект File, который содержит путь к файлу
   try {
       BufferedWriter bw = new BufferedWriter(new FileWriter(sdFile)); // открываем поток для записи
       bw.write(editTextEnter.getText().toString()); // пишем данные
       bw.close(); // закрываем поток
       Toast.makeText(this, "Файл записан на SD: "+ sdFile.getAbsolutePath(), Toast.LENGTH_SHORT).show();
       editTextEnter.setText("");
   } catch (IOException e) {
       Toast.makeText(this, "Не удалось записать файл", Toast.LENGTH_SHORT).show();
       e.printStackTrace();
   }
}
     При чтении файла, который находится на SD производятся почти аналогичные действия:
